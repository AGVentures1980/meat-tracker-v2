1
Day 1 — System Setup
Owner: Regional Director + GM
1.1
Create GM user in system Critical
Admin accesses Settings → Users → Add User. Role: Manager. Assign to correct store.
Admin: Settings → Users → Add User → Role: manager → Store: [store name]
1.2
Apply Operational Template Critical
Select the appropriate template for store profile. Consult Regional Director if unsure.
Settings → Company → Templates → [Select template] → Apply to store
1.3
Verify Price Configuration
Confirm Lunch and Dinner prices. Confirm if Lamb Chops are included in rotation.
Settings → Store → Edit → Prices + Lamb Chops
1.4
Test GM Login
GM logs in, confirms they see only their own store, and dashboard loads.
2
Days 2–3 — Initial Data Entry
Owner: GM (with Director support)
2.1
Enter Opening Inventory Critical
Weigh and record initial stock of all proteins. This is the system baseline.
Dashboard → Reports → Inventory → Opening Date → Enter quantities
2.2
Enter Weekly Prices
Enter cost prices for proteins for the opening week.
Prices → Weekly Price Input → Current Week → Fill and Save
2.3
Enter First Guest Report
Record guest count for the first week (estimate acceptable for opening).
Reports → Weekly Report → Opening Week → Lunch + Dinner Guests
3
Days 4–7 — Team Training
Owner: GM
3.1
GM completes Training Modules 1 & 2 Critical
Module 1: Reading the Dashboard. Module 2: Responding to Alerts. Quiz mandatory.
Training → Module 1 → Watch → Quiz → Module 2 → Watch → Quiz
3.2
Kitchen Team Briefing
Explain portioning protocol. Show what a "villain" is and why it matters.
3.3
Define Daily Data Entry Routine
Who enters data? When? Who is backup? Document internally.
4
Week 2 — Validation and Go-Live
Owner: Regional Director
4.1
Review Initial Data in Executive Summary Critical
Director verifies if store appears in network ranking. Does data make sense?
4.2
Confirm Alerts are Working
Simulate or await natural alert. Does GM know how to respond?
4.3
Sign Store Ready Checklist
GM and Director sign checklist. Store is officially live.
