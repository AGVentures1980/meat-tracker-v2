1
Dia 1 — Setup do Sistema
Responsável: Regional Director + GM
1.1
Criar usuário GM no sistema Crítico
Admin acessa Settings → Users → Add User. Role: Manager. Associar à loja correta.
Admin: Settings → Users → Add User → Role: manager → Store: [nome da loja]
1.2
Aplicar template operacional Crítico
Selecionar o template adequado ao perfil da loja. Consultar Regional Director se em dúvida.
Settings → Company → Templates → [Selecionar template] → Aplicar à loja
1.3
Verificar configurações de preço
Confirmar preços de almoço e jantar. Confirmar se Lamb Chops está incluso no rodízio.
Settings → Loja → Editar → Preços + Lamb Chops
1.4
Testar login do GM
GM faz login, confirma que vê apenas a própria loja, e que o dashboard carrega.
2
Dias 2–3 — Entrada de Dados Iniciais
Responsável: GM (com suporte do Director)
2.1
Inserir inventário de abertura Crítico
Pesar e registrar estoque inicial de todas as proteínas. Este é o baseline do sistema.
Dashboard → Reports → Inventory → Data de abertura → Inserir quantidades
2.2
Inserir preços semanais
Inserir preços de custo das proteínas para a semana de abertura.
Prices → Weekly Price Input → Semana atual → Preencher e salvar
2.3
Inserir primeiro relatório de guests
Registrar número de guests da primeira semana (estimativa aceitável para abertura).
Reports → Weekly Report → Semana de abertura → Guests almoço + jantar
3
Dias 4–7 — Treinamento da Equipe
Responsável: GM
3.1
GM completa Módulos 1 e 2 de treinamento Crítico
Módulo 1: Como ler o dashboard. Módulo 2: Como responder a alertas. Quiz obrigatório.
Training → Módulo 1 → Assistir → Quiz → Módulo 2 → Assistir → Quiz
3.2
Briefing da equipe de cozinha
Explicar o protocolo de porcionamento. Mostrar o que é "villain" e por que importa.
3.3
Definir rotina diária de entrada de dados
Quem insere os dados? Quando? Quem é o backup? Documentar internamente.
4
Semana 2 — Validação e Go-Live
Responsável: Regional Director
4.1
Revisar primeiros dados no Executive Summary Crítico
Director verifica se a loja aparece no ranking de rede. Dados fazem sentido?
4.2
Confirmar que alertas estão funcionando
Simular ou aguardar um alerta natural. GM sabe como responder?
4.3
Assinar Checklist Loja Pronta
GM e Director assinam o checklist. Loja está oficialmente no ar.
