Arquitetura de Zero-Fricção na Doca: Processo de Scan Único para Operadores de Chão de Fábrica

🎯

A Diretriz Não-Negociável

O operador de recebimento (chão de fábrica) deve realizar apenas um escaneamento por caixa e colocá-la na balança. Toda a complexidade lógica e validação tributária ocorre de forma silenciosa e invisível em background.

Etiqueta Branca Original

ROTA A: Scanner na Etiqueta do Frigorífico

Escaneia o código GS1-128 do frigorífico original (ex: JBS, Cargill) para ler o peso líquido exato direto da origem.

1

Escaneamento GS1-128

O operador faz um único scan na etiqueta do frigorífico. O BRASA extrai o GTIN e o peso real (ex: 63.6 Lbs).

2

Resolução EDI 856 / ASN

O sistema busca o GTIN/Serial no arquivo eletrônico pré-enviado pelo distribuidor para mapear o SKU automaticamente.

3

Inferência Biológica

Se não houver EDI, o algoritmo analisa a nota fiscal e associa o GTIN à carne com base na faixa de peso biológico em background.

Etiqueta Secundária

ROTA B: Scanner na Etiqueta do Distribuidor

Escaneia a etiqueta colada pela distribuidora (ex: Sysco, US Foods) para ler o SKU corporativo pré-mapeado do catálogo.

1

Escaneamento Local

O operador escaneia o código de barras local do distribuidor. O SKU (ex: Fraldinha) é resolvido de forma imediata.

2

Pesagem Obrigatória

O operador coloca a caixa na balança integrada. O peso físico real (ex: 63.6 Lbs) é capturado pelo sistema.

3

Quebra de Peso Fantasma

O BRASA compara a balança com o peso cobrado na NF (82.0 Lbs). Detecta o erro de 18.4 Lbs e abre disputa fiscal.

Engenharia de Background (Como a Mágica Acontece)

Mapeamento Invisível de GTIN

O algoritmo de inferência biológica elimina a necessidade de o funcionário selecionar a carne no tablet. Ele compara o peso real lido na etiqueta com as especificações cadastradas para os itens esperados na nota fiscal.

// Mapeamento Inteligente e Preditivo de Identidade const expectedProtein = invoiceItems.find(item => weightReal >= item.spec.minWeight && weightReal <= item.spec.maxWeight ); if (expectedProtein) { autoMapGtin(scannedGtin, expectedProtein.sku); }

Resolução de Wedge Scanners

Wedge scanners de teclado transmitem espaços, parênteses e caracteres de controle formatados. O BRASA normaliza a entrada crua antes do processamento lógico para evitar quebras sintáticas.

export function normalizeRawBarcode(raw: string) { const cleaned = raw.replace(/[\s\r\n\t\x1D\(\)\[\]]/g, '').trim(); return { cleaned, is_valid: /^[a-zA-Z0-9\-]*$/.test(cleaned) }; }

🛑 Regras de Interfaces do Operador de Chão de Fábrica

  • Sem Cliques Extras: Foco padrão permanente no leitor de código de barras. O funcionário não encosta na tela.
  • Sinais de Áudio Claros: Bipes curtos e agudos para aceitação (Verde); tons longos e graves para falha (Vermelho).
  • Visão Limpa: Ocultação completa de terminologias técnicas como hashes canônicos, queries SQL ou códigos API.
  • Conflitos Assíncronos: Qualquer dúvida de mapeamento de código é enviada silenciosamente para o painel administrativo do gerente.
