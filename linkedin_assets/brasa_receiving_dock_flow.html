Cenário A

Caixa #1: Aprendizado e Mapeamento de GTIN

Operação Primeiro Recebimento

FUXO FÍSICO

  • O operador coloca a primeira caixa do caminhão na balança (peso real de 63.6 Lbs).
  • Escaneia a etiqueta branca grande do frigorífico (USDA Packer).
  • Como o GTIN de 14 dígitos nunca entrou na loja antes, a tela entra em modo de mapeamento assistido.
  • O operador seleciona "Fraldinha Choice" na lista do tablet e clica em salvar.

EXECUÇÃO DO CÓDIGO

  • O `ReceivingController` recebe o scan e executa a decodificação GS1.
  • O `ComplianceEngine` retorna status `UNMAPPED_REVIEW_ALLOWED` devido ao GTIN inédito.
  • A rota `/review-map` cria a `receivingRecognitionRule` no banco de dados.
  • A caixa é salva na tabela `ProteinBox` vinculada permanentemente ao SKU Fraldinha.
Cenário B

Caixa #2: Recebimento Nativo Direto

Tempo Médio < 2 Segundos

FLUXO FÍSICO

  • O operador coloca a segunda caixa na balança (peso real de 64.1 Lbs).
  • Aponta o leitor e escaneia apenas a etiqueta branca grande do frigorífico.
  • A tela do tablet pisca verde e emite sinal de aprovação sonora.
  • O operador guarda a caixa na câmara fria. O processo segue rápido e livre de erros.

EXECUÇÃO DO CÓDIGO

  • O scanner envia a string crua para o backend.
  • O `BarcodeGS1Parser` localiza o GTIN `90076223888514` previamente mapeado.
  • O `ComplianceEngine` valida o peso de 64.1 Lbs dentro do limite biológico configurado (54-66 Lbs).
  • O banco grava a `ProteinBox` diretamente e gera o log histórico de ciclo de vida.
Cenário C

Caixa Replicada: Proteção de Estoque Duplo

Resultado Bloqueio Imediato

FLUXO FÍSICO

  • O operador se confunde na doca e tenta escanear novamente a Caixa #2 que já foi guardada.
  • O tablet emite sinal sonoro de falha e pisca a tela em vermelho com alerta de bloqueio.
  • O operador é forçado a verificar a caixa e o processo segue íntegro sem duplicações virtuais.

EXECUÇÃO DO CÓDIGO

  • A API recebe o código de barras e realiza uma busca de duplicidade na tabela `ProteinBox`.
  • O banco localiza um registro correspondente ativo para aquele código naquela loja.
  • A transação é rejeitada com código HTTP `409 Conflict`, travando a criação de dados espelhados.
Cenário D

Peso Fantasma: Fraude de Diferença Financeira

Recuperação por Caixa $167.25 USD

FLUXO FÍSICO

  • O distribuidor fatura a nota cobrando 82.0 Lbs por uma caixa (e cola a etiqueta Sysco amarela de 82.0 Lbs).
  • O operador coloca a caixa na balança física e lê o peso real de **63.6 Lbs**.
  • Ao escanear a etiqueta branca de frigorífico, o tablet detecta a quebra financeira.
  • O gerente aceita com ressalva: o estoque computa 63.6 Lbs e o BRASA gera a disputa de cobrança.

EXECUÇÃO DO CÓDIGO

  • A balança (63.6 Lbs) bate com a decodificação da etiqueta de frigorífico (63.6 Lbs).
  • O peso difere da Nota Fiscal importada (82.0 Lbs), excedendo o desvio máximo de umidade.
  • O `ComplianceEngine` gera uma quebra de conformidade e notifica o painel de auditoria.
  • O estoque computa apenas a entrada física real, gerando a divergência fiscal de 18.4 Lbs.

⚙️ Detalhes de Engenharia do Back-end

Decodificação de Strings GS1-128

O motor BRASA decodifica as strings com tratamento de expressões regulares sofisticadas e tolerantes, limpando formatações no normalizador e extraindo os pesos de forma precisa.

const lbWeightMatch = raw.match(/3201(\d{6})|3202(\d{6})/); if (weightLbMatch) { const raw6 = weightLbMatch[1] || weightLbMatch[2]; const decimals = weightLbMatch[1] ? 1 : 2; weightLb = parseFloat((parseInt(raw6, 10) / Math.pow(10, decimals)).toFixed(2)); }

Proteção de Entrada Limpa (Wedge Normalizer)

Normaliza caracteres para garantir que leitores virtuais e digitações de teclados emulados com espaços ou colchetes não causem quebras nas rotas do servidor.

export function normalizeRawBarcode(raw: string) { const cleaned = raw.replace(/[\s\r\n\t\x1D\x1E\x1F\(\)\[\]]/g, '').trim(); const is_valid_chars = /^[a-zA-Z0-9\-]*$/.test(cleaned); return { cleaned, is_valid_chars }; }
